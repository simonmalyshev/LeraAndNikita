# Подключение анкеты к Telegram без изменения старого сайта

Существующая Yandex Cloud Function сайта `Invintation` остаётся без изменений.
Для этого проекта создаётся **новая** функция с новым URL, но с тем же `BOT_TOKEN`
и пока с тем же `CHAT_ID`.

Новый обработчик унифицирован:

- новый сайт отправляет JSON через `POST` и получает сообщение в стиле «Лера и Никита»;
- обработчик также понимает старый JSONP-формат `Invintation` и формирует его прежний стиль сообщения;
- эта совместимость позволяет позднее перевести старый сайт на новую функцию, но сейчас этого делать не нужно.

## Что уже подготовлено

- Клиентская отправка всех полей анкеты: `js/main.js`.
- Унифицированная функция: `telegram-function/index.js`.
- Автотесты обоих форматов: `telegram-function/index.test.js`.
- Валидация, экранирование Telegram HTML, honeypot и проверка Origin.
- Раздельная маршрутизация получателей через `CHAT_ID` и `LERA_NIKITA_CHAT_ID`.

## 1. Не изменять старую функцию

Не редактируйте функцию с адресом:

```text
https://functions.yandexcloud.net/d4eeasrjvr6vjid7tkt0
```

Не меняйте URL в проекте `E:\Storage\Personal\Invintation`. Старый сайт продолжит
работать через свою существующую функцию независимо от нового проекта.

## 2. Создать новую Yandex Cloud Function

1. Войдите в Yandex Cloud.
2. Откройте Cloud Functions → «Создать функцию».
3. Назовите её, например `wedding-rsvp-unified`.
4. Создайте версию с runtime Node.js 18 или новее.
5. В редакторе создайте `index.js` и вставьте содержимое локального файла
   `telegram-function/index.js`.
6. Укажите точку входа:

   ```text
   index.handler
   ```

7. Дополнительные npm-пакеты и `package.json` не нужны.

## 3. Настроить существующего бота и текущего получателя

Скопируйте значения переменных из старой функции в новую через интерфейс Yandex Cloud:

| Переменная | Значение |
| --- | --- |
| `BOT_TOKEN` | Токен существующего Telegram-бота |
| `CHAT_ID` | Идентификатор текущего получателя |

Не переносите секреты в исходный код, Git или сообщения.

Пока новый сайт должен отправлять тому же человеку, переменную
`LERA_NIKITA_CHAT_ID` не создавайте. Обработчик автоматически использует `CHAT_ID`.

## 4. Опубликовать новую функцию

1. Создайте версию функции.
2. Разрешите публичный вызов: роль `functions.functionInvoker` для `All users`.
3. Публичный URL созданной функции:

   ```text
   https://functions.yandexcloud.net/d4e9i19l0j4qjmotejt6
   ```

4. Убедитесь, что этот URL отличается от URL старой функции.

## 5. Вставить новый URL в сайт

URL уже добавлен в `js/main.js`:

```javascript
var RSVP_ENDPOINT = 'https://functions.yandexcloud.net/d4e9i19l0j4qjmotejt6';
```

Не вставляйте URL старой функции.

## 6. Проверить локально

Автотесты функции уже проходят. Их можно повторить из корня проекта:

```powershell
node --test .\telegram-function\index.test.js
```

Затем запустите сайт:

```powershell
python -m http.server 8000
```

Откройте `http://localhost:8000`, заполните анкету и отправьте её. Текущий
получатель должен получить сообщение с заголовком «Новый ответ: Лера и Никита».

После этого отдельно проверьте старый сайт. Он должен продолжать работать через
старую функцию, которую мы не изменяли.

## 7. Ограничить браузерные источники после публикации

После появления окончательного домена добавьте к **новой** функции переменную:

```text
ALLOWED_ORIGINS=https://ваш-домен.ru,https://www.ваш-домен.ru
```

Адреса перечисляются через запятую без `/` в конце. До завершения тестирования на
`localhost` переменную лучше не задавать.

## 8. Позднее назначить нового получателя только этому сайту

1. Новый пользователь открывает того же Telegram-бота.
2. Нажимает Start или отправляет `/start`.
3. Получите его `chat_id` через метод Telegram `getUpdates`.
4. Добавьте к **новой** функции:

   ```text
   LERA_NIKITA_CHAT_ID=<chat_id нового пользователя>
   ```

5. Опубликуйте новую версию и отправьте тестовую анкету.

Результат:

- `LeraAndNikita` отправляет новому пользователю;
- `Invintation` продолжает работать со старой функцией и старым получателем;
- бот остаётся общим.

Чтобы вернуть текущего получателя, удалите `LERA_NIKITA_CHAT_ID` из новой функции
и опубликуйте ещё одну версию.

## Возможная миграция старого сайта в будущем

Новая функция уже поддерживает прежние параметры `name`, `status`, `comment` и
JSONP callback. Поэтому позднее можно направить `Invintation` на новый URL без
изменения формы. Это отдельная операция: выполнять её сейчас не требуется.

## Диагностика

- **Форма пишет, что не настроена:** проверьте URL новой функции в `js/main.js`.
- **HTTP 403:** проверьте публичную роль функции и `ALLOWED_ORIGINS`.
- **HTTP 500:** проверьте `BOT_TOKEN`, `CHAT_ID` и журналы новой версии.
- **Telegram ничего не прислал:** проверьте, что пользователь уже запускал бота.
- **Старый сайт сломался:** новая функция не является причиной, если URL и старая
  функция действительно не изменялись.
